### Sprint 3:Embedding root cause + fix

挖到 root cause:
- OpenAI `text-embedding-3-small` max 8192 tokens per input
- 既有 `.slice(0, 8000)` 中文 1 char ≈ 1 token 會超
- **整 batch (100 chunks) 因 1 個過長 chunk 全失敗** 400 invalid_request
- 過去幾週 embedding cron 每次跑都 partial 0 processed,524 chunks 不斷累積

修 4 處(`9ce5590` + `6bcff06`):
- `src/app/api/cron/embedding-refresh/route.ts`
- `src/app/api/admin/rag/embed-pending/route.ts`
- `src/app/api/claude-panel/chat/route.ts`
- `src/app/api/rag/search/route.ts`

`slice(0, 8000)` → `slice(0, 4000)`(中文 4000 tok 留 headroom)。

寫 `embedding-flush.yml` workflow 跑 4 次清 backlog:
- Iter 1: 200 processed
- Iter 2: 200 processed
- Iter 3: 124 processed
- 共 524/524 全清空 ✅

Re-audit 確認:`knowledge_chunks_pending_embedding: 0`,`with_embedding: 629/629 (100%)`。

### Sprint 4:Legacy cron deprecate

xuemi 66% 失敗 root cause(透過 sample_errors 抓到):
```
"METABASE_BLOCK: 缺 METABASE_USER / METABASE_PASS env vars"
```

差異:Q1381 走 GitHub Actions workflow 用 secrets refresh Supabase session cache 每 15 min,session 永遠 fresh。Legacy `/api/cron/metabase-sync` 走 `getMetabaseToken()` cache,過期就 fall through 到 `process.env.METABASE_USER` → Zeabur env 沒 → fail。

判斷 B 派:Q1381 cross-brand 已 supersede 各 brand sync,legacy 是冗餘。

修(`6c50409`):
- `/api/cron/metabase-sync` 改回 `410 Gone` + `deprecated: true` JSON
- 不再 iterate enabled brands 不再 log 噪音

`/api/admin/metabase-sync`(管理員手動 trigger)還動的,只 deprecate cron。
