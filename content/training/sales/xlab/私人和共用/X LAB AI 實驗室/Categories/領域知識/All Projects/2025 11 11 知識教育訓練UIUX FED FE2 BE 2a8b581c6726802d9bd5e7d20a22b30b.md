# 🧩 一、介面設計 ( UI/UX ) 是什麼？

想像你正在使用一個 App 或網站：

📱 操作起來順不順手？

🖱️ 按鈕好不好找？

🧭 能不能很快找到你想要的資訊？

這些背後都靠 **UI/UX 設計**。UI/UX 就像是一位「幫你把產品做得更好用」的設計師，透過 **FIGMA** 的軟體設計讓使用者看到介面時覺得舒服、使用時覺得順手，避免操作卡住、流程繁瑣、找不到按鈕等狀況發生。

# 🧠 二、它是怎麼運作的？

在 UI/UX 裡，我們會看到很多流程階段，例如：

- 🎯 使用者調查（User Research）
- 🤔 問題定義（Problem Statement）
- 🧱 線框稿（Wireframe）
- 🎨 介面視覺設計（UI Design）
- 🧪 可用性測試（Usability Test）

UI/UX 設計師做的事，像是在「打造一段使用者旅程」。

從理解需求 → 規劃流程 → 做出介面 → 測試 → 上線。**畫遊樂園地圖的人，決定溜滑梯在哪、門口怎麼走、排隊路線怎麼設計。**

# ⚙️ 三、舉幾個超簡單的例子

| 情境 | UI/UX 能幫你做的事 |
| --- | --- |
| 會員註冊流程太複雜 | 設計更直覺的 3 步驟註冊流程 |
| 購物車放棄率高 | 調整右上角 CTA、簡化結帳介面 |
| 客服常收到「找不到按鈕」 | 重新規劃資訊架構、提升導航清晰度 |
| 使用者抱怨頁面太難讀 | 改善視覺層級、間距、字體、排版 |
- 你**不需要會寫程式**即可入門 UI/UX。
- 優秀的 UI/UX 能讓產品轉換率提升、留存變高、客服量下降。
- 現在企業高度依賴 UI/UX：
    
    電商、醫療、金融、教育、App、系統、SaaS…幾乎無所不在。
    

# 📘四、快速了解手冊

> 好的介面不是更花俏，而是更貼近人性。
> 

UI/UX 是「**使用者介面 + 使用者體驗**」的整合設計流程。

**UI（User Interface） → 負責介面、顏色、字體、按鈕、版面。**

**UX（User Experience） → 負責流程、邏輯、行為、動線、體驗。**

> 「理解使用者問題 → 做出好理解、好用、好看的產品」你不需要寫程式，但要能用工具（如 Figma）打造專業介面。
> 

| 功能價值 | 說明 |
| --- | --- |
| 🔍 理解使用者 | 用訪談、問卷、測試找出真正需求 |
| 🧭 規劃清晰動線 | 減少迷路、降低使用門檻 |
| 🎨 介面更美更一致 | 提升品牌信任感與專業度 |
| 📊 降低營運成本 | 客服問問題變少、錯誤操作降低 |
| 📱 全平台一致 | Web、Mobile、系統后台都能統一體驗 |

# 🧩 一、前端設計 ( FED ) 是什麼？

前端設計是介於「設計稿」與「程式碼」之間的角色

做能直接落地的設計（含 HTML/CSS 原型）它是「美感」＋「程式理解」的混合角色，比 UIUX 設計更需要實作能力。如同幫溜滑梯上色、把樂園變漂亮，確保小朋友看到就想玩，而且走起來順順的。

# 🧠 二、它是怎麼運作的？

前端設計師通常會：

- 使用 Figma 做設計
- 使用 HTML / CSS 打出可用原型
- 用 SCSS 調整排版 + 做細緻動畫（Motion UI）
- 與前端工程合作交付高品質畫面

介面不只是漂亮，而是：

- 有系統
- 有排版邏輯
- 可被工程端採用無縫落地

# ⚙️ 三、舉幾個超簡單的例子

| 情境 | 前端設計能做的事 |
| --- | --- |
| 想做漂亮的課程頁面 | 完成設計圖 → 實作 HTML+CSS → 給工程師串接 |
| 需要高質感動態頁面 | 做 micro-interaction 動畫 |
| 需要能套用的設計系統 | 打造 Design System + Component Library |
| 需要快速做出 Demo | 產出可點擊＋可展示的前端原型 |

前端設計的強大之處在於：

👉 設計不是畫圖，而是可以直接跑起來。

👉 工程拿到後不用重切版，效率倍增。

# 🧩 一、前端工程 ( FE2 ) 是什麼？

當你打開一個網站或 App：

👉 按鈕會動

👉 視窗會彈出

👉 表單會驗證

👉 整個畫面會流暢切換

這些互動功能，都是 **前端工程師** 做的。

前端工程就像是「把設計變成真正可操作的產品」，負責讓畫面跑起來、功能能互動、資料能傳送到後端。把溜滑梯、旋轉木馬做出來的人，讓小朋友真的可以玩、按按鈕會動。

# 🧠 二、它是怎麼運作的？

前端工程的世界由三大核心技術組成：

- 🧱 **HTML**：負責結構
- 🎨 **CSS**：負責外觀
- ⚡ **JavaScript**：負責動態功能

再加上：

- 🔧 前端框架：React、Vue

前端工程師的任務像是：

把一張設計稿 → 做成一個真正能用的網站或功能。

# ⚙️ 三、舉幾個超簡單的例子

| 情境 | 前端工程能幫你做的事 |
| --- | --- |
| 要做一個報名表單 | 做出視覺介面＋即時驗證＋串後端 API |
| 要做儀表板 | 讓圖表動起來＋自動更新資料 |
| 要做產品官網 | 讓動畫、動線、排版都能動 |
| App/網站要串 ChatGPT | 加入 prompt 欄位、回覆 UI、資料傳輸 |
- 前端工程讓畫面「真的運作」
- 負責互動、動態、資料交換
- 是產品與使用者的交互橋樑

# 📘 四、快速了解手冊

# 🧩一、後端工程 ( BE ) 是什麼？

當你使用一個網站／App：

✔ 你按下「登入」按鈕

✔ 你送出訂單

✔ 你查詢交易紀錄

✔ 你瀏覽課程進度

背後所有「看不到」的邏輯、資料儲存、驗證、計算、流程

通通由 **後端工程師**負責。

後端就像一個巨大的機房與大腦：

👉 處理資料

👉 控管權限

👉 計算結果

👉 保證資料安全

👉 決定每個功能「怎麼跑、跑去哪、跑對不對」

如果前端像是溜滑梯、旋轉木馬，**後端就鋪電線、水管、控制房、機器室的人，讓溜滑梯不會壞、旋轉木馬一直能運作。**。沒有後端，產品「只能看，不能用」。

# 🧠 二、它是怎麼運作的？

後端工程圍繞在「資料」與「邏輯」：

- 🗄️ **資料庫**：MySQL、PostgreSQL、MongoDB
- 🔌 **API**：REST、GraphQL
- 🏗️ **後端框架**：Node.js、Django
- 🛠️ **雲端**：AWS、GCP
- 📦 **部署與維運**：Docker、CI/CD

你每天使用的 App／網站，都會跟後端 API 對話。

前端送出資料 → 後端處理 → 回傳結果給前端 → 顯示給使用者

後端工程師＝處理資料 + 保證系統安全 + 確保產品正常運作的人。

# ⚙️ 三、舉幾個超簡單的例子

| 情境 | 後端工程能幫你做的事 |
| --- | --- |
| 會員登入 | 建立帳號 → 加密密碼 → 驗證權限 |
| 電商結帳 | 訂單成立 → 扣庫存 → 開發票 → 寄信通知 |
| 訊息系統 | 儲存聊天室紀錄 → 推播 → 既讀同步 |
| 課程系統 | 紀錄進度 → 計算學習時數 → 顯示統計圖 |

後端負責：

- 做資料
- 做邏輯
- 做安全
- 做運算
- 做 API
- 做管理後台

簡單說：

👉 **前端負責讓你看得見 / 點得到**                👉 **後端負責讓一切真正運作起來**

# 📘 四、快速了解手冊

> 後端不是寫資料庫，而是打造整套「系統的運作核心」。
> 

後端工程專注於：

- 開發 API
- 定義資料結構
- 設計系統流程
- 處理運算
- 資安與權限
- 讓整個服務穩定運作

你看不到後端的畫面，但只要後端掛掉，整個產品就會停止運作。

| 功能價值 | 說明 |
| --- | --- |
| 🗄️ 資料處理 | 負責 CRUD、查詢、計算、儲存 |
| 🔐 系統安全 | 防止駭客、SQL 注入、權限漏洞 |
| ⚙️ 業務邏輯 | 負責系統規則、流程判斷 |
| 📡 API 溝通 | 讓前端能取得正確資料 |
| 🧱 架構設計 | 讓系統可長期維護與擴充 |
| 🚀 效能優化 | 加快查詢速度、降低壓力 |
| 📊 系統穩定性 | 維運、監控、日誌、故障自動修復 |
