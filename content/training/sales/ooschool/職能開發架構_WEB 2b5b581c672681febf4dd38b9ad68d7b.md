# 職能開發架構_WEB

文字: 職能開發架構包含背景探索、技能盤點、價值塑造與市場需求分析，強調實戰學習與個性化課程，並提供免費諮詢以協助職涯轉換或接案規劃。

```jsx
|- 開場
   |- 「哈囉！請問是___嗎？我是X平台的顧問___，有看到你有留資料，是有在關注網站前端、後端開發相關的內容嗎？」
   |- 「目前是有希望做職涯轉換，還是主要是幫自己增強技術能力呢？」

|- 初步背景探索
   |- 詢問現職：
      |- 問題：「你現在在做什麼工作啊？」
      |- 得知：學生 / 上班族 / 自由工作者 / 剛離職
   |- 詢問學習動機：
      |- 問題：「是希望轉職當工程師？還是未來想接案、做自由接案？」
      |- 得知：想轉職 / 技術升級 / 接案用 / 創業需要

|- 技能盤點（依照網站開發領域）
   |- 詢問前端經驗（HTML/CSS/JS）
      |- 得知：有基礎 / 完全新手 / 看過但不會實作
      |- 延伸引導：
         |- 「你有沒有試過自己做一個個人網站？」
         |- 「我們有學員從0開始練切版，結果被設計公司直接找去面試！」

   |- 詢問框架與互動設計（React/Vue等）
      |- 得知：聽過但沒學 / 正在自學 / 想做更完整的動態網站
      |- 延伸引導：
         |- 「你有寫過像Todo List這種互動功能嗎？」
         |- 「有一位學員兩個月內完成互動式預約系統，還直接幫老闆升級了公司網站！」

   |- 詢問後端經驗（Python/Node/Flask）
      |- 得知：只會前端 / 有學過Python但不會串資料庫
      |- 延伸引導：
         |- 「有想過自己做一個登入系統或會員後台嗎？」
         |- 「有學員寫一個訂單管理系統給接案用，後來變成自己創業的後台基礎！」

   |- 詢問資料庫/部署/Git經驗
      |- 得知：不太熟 / 曾用過Firebase或MySQL
      |- 延伸引導：
         |- 「如果今天要做一個可以記帳的系統，你知道要怎麼存資料嗎？」
         |- 「很多人卡在資料庫串接，我們有完整從資料流設計到實作的教學。」

|- 概念釐清與價值塑造（教學階段）
   |- 解釋網站開發是什麼：
      |- 「簡單來說，前端就是把畫面做出來，後端是讓功能動起來。」
      |- 「像你打開一個購物網站，能看到的畫面是前端，加入購物車、登入、付款這些邏輯就是後端。」
   |- 延伸價值說明：
      |- 「網站開發人員在市場上一直都有需求，尤其是全端能力越來越搶手。」
      |- 「有些人是做接案，有些人會進科技新創或設計公司。職涯彈性蠻大的。」
   |- 案例舉例：
      |- 「我們有一位非本科轉職的學員，從練習做個人作品集開始，半年後被一間新創公司錄取，現在負責整個官網維護與改版。」
      |- 「還有學員原本是美術設計背景，後來跨足前端設計，結果拿到30萬起薪offer。」

|- 市場價值與接案價值強調
   |- 「其實很多公司現在找人，不是只會畫面，也希望你懂怎麼串API、怎麼設計後台邏輯。」
   |- 「我們也有很多學員後來成為freelancer，一個網站接案從2萬到10萬都有，尤其你如果設計感也不錯會更吃香。」
   |- 「所以我們整個教學會把技能打包成完整職能，一條龍教你從切版到資料庫再到部署。」

|- 方案升級與藍圖描繪
   |- 整合學員技能與目標：
      |- 「你會一點前端，代表你不是從0開始，我們就可以直接進入實作和實戰的專案模組，加速學習時間。」
   |- 提出藍圖：
      |- 「你如果每週可以投入8-10小時，差不多3-6個月內就能做出能用的作品，像購物網站、會員系統、個人作品集等等。」
      |- 「這些都是可以直接投履歷、或拿來接案用的東西。」

|- 課程模式與教學差異化說明
   |- 強調「不是影片課」、「不是填鴨制」：
      |- 「我們是業界唯一教練＋總監雙指導的教學，真的有人會陪你做、幫你改、給你專案審核建議。」
   |- 引入X Platform 教學特色：
      |- 「四大團隊會協助你從課程顧問（幫你規劃學習進度）、無敵助教（行政文件協助）、學習教練（技術教學）、資深總監（專案方向審核）」
      |- 「不是只有教你寫，而是會帶你做、陪你討論、讓你改到懂為止。」
   |- 強調完整實戰里程：
      |- 「整個教學時數從120HR~210HR，包含前端、後端、框架、資料庫、Git、部署。」
      |- 「結訓後你會有完整的作品集、GitRepo、能操作的產品原型。」

|- 預算與價格確認
   |- 問題：「你目前對這樣的課程大概有預算上的初步概念嗎？例如一次付完還是會想分期？」
   |- 得知：有概念 / 想分期 / 預算不高
   |- 引導：
      |- 「我們方案會依照你現有技能、學習節奏做客製化的安排，不會讓你花冤枉錢。」
      |- 「也有分期方案配合，像學員方案約落在4~5萬起跳，完整方案則會更完整含專案與後端模組。」
      |- 「你現在不用決定，我們會在下一步諮詢中幫你完整規劃，再看是否需要做調整。」

|- Call To Action（主要行動呼籲）
   |- 「不然這樣好了，我們安排一個免費諮詢，是由總監親自幫你看背景，順便教你如何從學習轉成接案或轉職履歷規劃。」
   |- 「這一段時間會用畫面螢幕分享，教你怎麼看出自己的市場價值。」

|- 預約與收尾
   |- 「那你明天晚上7點有空嗎？我們先安排30分鐘諮詢時間。」
   |- 「等等我會發官方LINE的加入連結給你，我們會發資料、學員作品案例給你看一輪。」
   |- 「你可以稍微看一下，明天就能更深入聊囉！」

```
