# 職能開發架構_PRD

文字: 職能共學平台針對UIUX轉職及接案提供專業腳本，涵蓋學習動機、核心技能、產品規劃經驗及市場行情，並強調實戰教學與個別指導，幫助學員成功轉職或接案。

```jsx
|- PRD產品設計專屬業務腳本

   |- 初步背景探索

      |- 詢問現職
         |- 問題：你好，我這邊是X Platform職能共學平台，請問是 xxx 先生嗎?
         |- 客戶回應
         |- 我們在(素材來源)，看到您想要了解UIUX轉職和接案以及數位游牧的資訊->接下去延伸引導的問題
         |- 延伸引導：
            |- 學生：現在很多設計本科學生都會學 Figma 或 UX 流程，畢業比較好找產品實習。
            |- 上班族：有些產品經理是從客服或行銷轉來的，因為熟用戶需求，會畫流程圖就能轉職。
            |- 自學者：像有學員自己做 side project，結果靠 Notion 流程圖＋Figma mockup 拿到 offer。
            |- 待業：想不想趁這段時間先學會用 Figma 跟建立自己的作品流程？

      |- 詢問學習動機
         |- 問題：你會想學產品設計，是因為哪一塊最吸引你？想轉職？還是做 side project？
         |- 延伸引導：
            |- 想轉職：有些人從客服或網拍助理轉職產品設計，一開始都從學 UX 流程開始。
            |- 想接案：我們有學生專接「流程優化＋原型製作」案子，一次收案就有上萬。
            |- 想加值：不一定要當 PM，行銷或 BD 會懂 PRD 和流程畫圖，也很加分。

   |- 核心技能盤點

      |- 詢問UX流程熟悉度
         |- 問題：你知道 UX 流程嗎？像是 persona、用戶旅程地圖、flow chart、wireframe？
         |- 得知狀況：
            |- [得知: 有學過UX流程，但沒有實際做過完整專案]
            |- [得知: 聽過名詞但流程卡住，不知道怎麼開始]
            |- [得知: 完全新手，但聽說UX是熱門轉職領域]
         |- 延伸引導：
            |- 「像蝦皮購物結帳流程的每一個步驟，背後其實都有做過 UX 分析和測試。」
            |- 「我們有學員幫朋友咖啡廳做會員系統的 UX 設計，還自己跑 user test 拿下專案！」

      |- 詢問UI工具使用狀況
         |- 問題：你有用過 Figma 或 Adobe XD 畫過 wireframe 或 prototype 嗎？
         |- 得知狀況：
            |- [得知: 有用過 Figma 畫畫玩，沒做過完整設計]
            |- [得知: 有做過作品，但對元件管理和 auto layout 不熟]
            |- [得知: 沒碰過工具，只用過 Canva 或簡報]
         |- 延伸引導：
            |- 「像現在面試 UIUX 幾乎都要會 Figma，還要能做 prototype 給開發或提案用。」
            |- 「有學員原本只會用 Canva，現在會畫 prototype，幫補習班做報名系統直接接案。」

      |- 詢問產品規劃/PRD經驗
         |- 問題：你有聽過 PRD 嗎？或寫過類似的產品需求文件？
         |- 得知狀況：
            |- [得知: 有接觸 PM 工作，但沒正規寫過 PRD]
            |- [得知: 自己學 Notion 記錄產品想法，但沒跟工程師協作過]
            |- [得知: 完全沒碰過，但對產品開發流程很有興趣]
         |- 延伸引導：
            |- 「PRD 就像工程的藍圖，一份寫得好，設計和工程才會知道要做什麼。」
            |- 「有學員做了一個線上預約系統的 PRD＋流程圖，後來丟到 Discord 社群直接被找去當協作夥伴！」
      |- 詢問跨部門協作經驗
         |- 問題：你有跟工程或設計師合作過嗎？或是有協作經驗？
         |- 得知狀況：
            |- [得知: 曾參與新創提案或學校專案]
            |- [得知: 沒正式合作過，但想了解產品怎麼從0到1]
            |- [得知: 完全沒經驗，但常用app時會注意它哪裡做得好]
         |- 延伸引導：
            |- 「有些人一開始不是學設計，是從觀察產品痛點開始，慢慢轉成產品思維。」
            |- 「我們有學生參加 Shopee 類比賽，組隊練習PM、設計、工程協作，最後還得獎！」

   |- 價值釐清與市場行情補充
      |- PRD/UX接案行情：
         |- 「UX流程圖＋Figma原型接案一次平均5000~20000，還有些是週期性合作。」
         |- 「Notion + Figjam＋Figma整套流程導入，設計顧問一套要價3萬～5萬不等。」
      |- 職缺價值差異：
         |- [產品設計] 平均薪資 35K~60K，設計能力強＋懂流程可達 70K
         |- [UX設計師] 多為科技/新創公司職缺，常與PM工程合作，平均薪 45K~80K
         |- [產品經理PM] 如會PRD撰寫、流程導圖、能跨部門溝通，薪資從55K起跳，高可至120K以上
      |- 補充說明：
         |- 「PRD 的能力不是只有 PM 要會，很多 UX 設計師也都會畫流程圖，甚至帶 Sprint。」
         |- 「現在很多公司喜歡找 T 型人才，不只會畫介面，還能規劃產品流程和資料結構。」

   |- 概念釐清與價值塑造（教學階段）
      |- 解釋產品設計是什麼：
         |- 「簡單來說，產品設計是讓使用者順順地用完一個功能、完成任務。」
         |- 「像你打開 Uber 點餐，從定位、選擇、結帳，全程沒卡住，就是因為背後 UX / UI 設計到位。」
      |- 延伸價值說明：
         |- 「現在很多公司會要 UI/UX 能力，因為一個順暢的流程，可以直接降低客服量、提升下單率。」
         |- 「產品設計師在公司內的角色越來越重，是業務與技術之間的橋樑。」
      |- 實務比喻：
         |- 「有點像你開 Google 地圖，它不會問你很多話，直接幫你走到目的地。這就是產品流程設計。」
      |- 案例舉例：
         |- 「我們有一位學員原本只是幫自己做一個記帳工具 prototype，後來給老闆看，被拉進正式案子中做UX流程。」
         |- 「也有人是幫補教業畫報名流程，從人力填表變成線上報名系統，流程優化一次賺到 2 萬。」
         
   |- 方案升級與學習藍圖描繪
      |- 整合技能：
         |- 「剛剛聽下來你其實有設計底，只是 UX 還沒建完整流程，那這邊就可以把你往產品規劃升級。」
         |- 「你如果願意學會用 Figma + Notion + PRD 文件整合，可以從設計師往產品規劃走。」

      |- 未來藍圖描繪：
         |- 「我們的學員學會產品流程之後，不只轉職成功，也有很多人用這一套流程去做副業或幫公司升級流程，價值感差很多。」

   |- 課程制度與模式說明
      |- 差異化說明：
         |- 「這邊不是錄影帶型的教學，我們是『教練制』+『實戰專案』＋『一對一批改』。」
         |- 「你每週會安排教練會議，搭配階段任務跟個人作品實作，最後還能練 pitch。」
      |- 學習時程引導：
         |- 問題：你現在每週可以安排多少時間給自己進修呢？
         |- 引導：其實每週抓 6～8 小時，就可以在 3～5 個月內完成核心訓練＋專案實戰。

   |- 預算討論與報價確認
      |- 問題：你自己有預設過多少預算學這個領域嗎？
      |- 引導：
         |- 「我們這邊 PRD 系列完整方案包含教練＋3 階段專案，目前價格大約在 48,000 上下（含講師顧問時間與實作）。」
         |- 「你如果是目標轉職或接案，我可以幫你安排更彈性的付款方案，像是分期、或先從小模組開始。」
         |- 「而且我們跟一般單一講師教學不同，是四大團隊一起帶的：包含課程顧問幫你定目標、無敵助教幫你行政處理、學習教練每週陪你做作業、還有資深總監定期幫你審查專案方向。」
         |- 「整個教學是『雙指導制』，除了教練會帶你做，總監也會幫你審方向和設計方法，讓你做的不是作業，而是業界真的能用的專案。」
         |- 「我們每一梯學員都要完成 2～3 件完整專案＋Pitch＋履歷建立，整體時數在 120～210 小時之間，是業界唯一從流程、規劃、到實作全陪跑的訓練。」
         
   |- 行動呼籲與預約安排
      |- 試探 CTA：
         |- 「如果你有興趣，我這邊可以開一個免費螢幕分享，讓你看看學員的實際作品，還有規劃教案的內容。」
         |- 「像我們最近才剛結束一梯的 UX 設計挑戰賽，裡面還有參賽作品可以一起看。」
      |- 客戶同意：
         |- 「你這禮拜有空的時間大概是什麼時段呢？晚上 8-10 點之間哪一天比較可以？」
         |- 「那我先幫你預約明天晚上8點，會由我們負責PRD方向的資深顧問幫你規劃方向與進度。」

   |- 收尾與關係建立
      |- 建立聯繫：
         |- 「我先邀你加入我們官方 LINE 帳號，等一下發你會議連結＋資料 PDF。」
         |- 「你可以先看看學員的 UX 專案流程圖，大概就知道我們訓練強度與成果有多實戰了！」
      |- 提前預告：
         |- 「明天會議時間大概 30 分鐘左右，你可以事前準備一個你自己感興趣的 APP 或流程，我們可以直接幫你分析。」
      |- 總結提醒：
         |- 「那我這邊先設定好預約囉，等一下會傳你一份 UX 專案分析小測驗，幫你暖身一下，明天見！」
```
