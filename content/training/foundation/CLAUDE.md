## 📝 各檔案產出規範

### `script.md`（影片腳本）
必須包含：
- **單元基本資訊**（單元代號、時長、角色）
- **開場鉤子**（15 秒內抓住注意力）
- **分段式腳本**：每段含 `[場景]`、`[角色]`、`[旁白/對白]`、`[畫面建議]`、`[字幕]`
- **結尾呼應**（回到開場，或下一個行動）

範本段落：
```markdown
## 第 1 段（0:00–0:30）— 開場

[場景] 辦公室，小美剛入職
[角色] 小美（新人）
[對白] 欸我想請假但完全不知道從哪按耶...
[畫面] 小美對著電腦一臉困惑
[字幕] 第一次請假？看完這支影片你就不會慌了
```

### `interactive.html`（互動網頁）
- **單檔**，CSS/JS 內嵌，不依賴外部 CDN（平台環境可能無法訪問）
- **無 localStorage、sessionStorage**（平台可能封鎖）
- **響應式設計**（手機、電腦都要能看）
- **無障礙性**：鍵盤可操作、對比度足夠
- **至少 3 題**互動（題型見 Spec Section 7）
- **結尾** 必須有「完成學習」的明確訊號（給平台追蹤用）

### `handbook.md`（文件手冊）
- 標準 Markdown，可轉 PDF
- 含目錄（H2 以上自動生成）
- 規章條文用表格或 blockquote
- 結尾附「常見問題 FAQ」

### `README.md`（單元說明）
固定結構：
```markdown
# {單元代號} {單元名稱}

## 學習目標
- ...

## 受眾
- ...

## 時長
- 影片：X 分鐘
- 互動：X 分鐘
- 文件：X 分鐘閱讀

## 前置單元
- ...

## 評量通過標準
- 互動網頁答對 X/Y 題
```

## 🎨 互動網頁的視覺規範

統一風格讓平台內所有單元看起來像一家人：

- **主色**：深藍 `#1E3A5F`（專業感）
- **輔色**：暖橘 `#F59E0B`（重點/CTA 按鈕）
- **答對**：綠 `#10B981`
- **答錯**：紅 `#EF4444`
- **背景**：白 `#FFFFFF` 或淺灰 `#F9FAFB`
- **字體**：系統預設（`-apple-system, "Segoe UI", "Noto Sans TC", sans-serif`）
- **圓角**：`border-radius: 8px`
- **陰影**：`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`
