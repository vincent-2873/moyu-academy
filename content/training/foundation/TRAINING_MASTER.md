> **Claude Code 每次啟動任務前，必須先讀取本文件與對應的 Module Spec。**

## 1. 訓練體系總覽

### 1.1 公司規模與對象
- 員工規模：50–200 人
- 訓練對象：全體內部員工
- 消費方式：**公司內部平台播放**（非對外、非下載）

### 1.2 三大訓練體系

| 體系代號 | 中文名稱 | 主要受眾 | 規格書 |
|---------|---------|---------|--------|
| HR | 人資訓練 | 全員 / 主管 / 新人 | `HR_MODULE_SPEC.md` |
| BIZ | 業務訓練 | 業務部、客服 | `BIZ_MODULE_SPEC.md` |
| LEGAL | 法務訓練 | 全員（合規）/ 法務部 | `LEGAL_MODULE_SPEC.md` |

**本階段優先處理：HR 人資訓練。** 業務與法務體系在 HR 跑通後再複製流程。

## 2. 內容形式規範（混合三型）

由於員工在平台內播放消費，所有單元預設同時產出三種形式，讓員工可以依情境選擇：

| 形式 | 適用情境 | 工具 | 時長建議 |
|------|---------|------|---------|
| **A. 影片** | 觀念講解、情境演繹 | HeyGen / Hyperframes | 3–5 分鐘 |
| **B. 互動網頁** | 分支選擇、情境判斷、測驗 | HTML + JS（Claude Code 產出） | 5–10 分鐘 |
| **C. 文件手冊** | 查閱、複習、規章細則 | Markdown → PDF | 不限 |

**規則：**
- 每個單元必須至少產出 A + B，C 視主題決定
- 三種形式**內容一致、調性一致**，不可互相矛盾
- 影片負責「感覺」，互動網頁負責「練習」，文件負責「查詢」

## 3. 品牌調性與用字規範

### 3.1 整體調性
- **專業但不死板**：像一個有經驗的同事在帶你，不是教科書
- **具體不抽象**：多用情境、案例、數字；少用「我們應該要」這種空話
- **簡潔不廢話**：一句話能講完就不用兩句

### 3.2 用字原則
- 稱呼員工：「你」（不用「您」，太疏遠）
- 稱呼公司：「公司」或直接用公司名（不要用「本公司」）
- 避免翻譯腔：不要「做出一個決定」，直接說「決定」
- 中英混用 OK，但技術術語第一次出現要附中文說明

### 3.3 禁用詞 / 禁用模式
- ❌ 不要「首先...其次...最後...」這種教科書結構
- ❌ 不要 emoji 當項目符號（平台播放不一定支援）
- ❌ 不要出現真實員工姓名（情境案例用虛構角色）
- ❌ 不要有「本課程結束」這種尾巴，直接結尾就好

## 4. 固定角色人設（貫穿所有體系）

為了讓員工有熟悉感，所有影片使用**同一組虛擬講師**：

### 主講師：阿凱（KAI）
- 角色定位：訓練長 / 資深前輩
- 個性：專業、直接、偶爾講個冷笑話
- 用途：觀念講解、制度說明、重點複習
- HeyGen Avatar 編號：（待建立後填入）

### 未來可擴充角色（需要情境劇時再啟用）

目前 HRBP 系列 EP1-4 僅用阿凱單角主講 + Vincent/Lynn 原聲片段，未用到下列情境角色。


## 5. 檔案與命名規則

### 5.1 專案資料夾結構
```
training/
├── TRAINING_MASTER.md           # 本檔
├── specs/
│   ├── HR_MODULE_SPEC.md
│   ├── BIZ_MODULE_SPEC.md       # 之後擴充
│   └── LEGAL_MODULE_SPEC.md     # 之後擴充
├── templates/
│   └── UNIT_INPUT_TEMPLATE.md
├── units/
│   └── HR/
│       ├── HR-001-onboarding/
│       │   ├── input.md          # 填好的模板
│       │   ├── script.md         # 影片腳本
│       │   ├── interactive.html  # 互動網頁
│       │   ├── handbook.md       # 文件手冊
│       │   └── assets/           # 圖片、配音等
│       ├── HR-002-.../
│       └── ...
└── build/                        # Claude Code 產出區
```

### 5.2 命名規則
- 單元代號：`{體系}-{三位數}-{英文關鍵字}`
  - 範例：`HR-001-onboarding`、`HR-002-leave-policy`
- 檔名全小寫、連字號分隔，不要空格、不要中文檔名
- 版本控制：大改版用 `-v2`、`-v3` 結尾

## 6. Claude Code 執行守則（防跑偏關鍵）

> **這段是給 Claude Code 看的，請嚴格遵守。**

### 6.1 每次任務啟動流程
1. **先讀** `TRAINING_MASTER.md`（本檔）
2. **再讀** 對應的 `{體系}_MODULE_SPEC.md`
3. **最後讀** 當前單元的 `input.md`
4. 三份文件若有衝突，**以上層為準**（Master > Spec > Input）

### 6.2 不可自由發揮的項目
- ❌ 不可新增未在 Spec 中定義的主題
- ❌ 不可改變角色人設
- ❌ 不可改變檔案命名規則與資料夾結構
- ❌ 不可輸出本文件禁用的用字與格式
- ❌ 不可產出 Spec 未要求的檔案類型（例如擅自做 PPT）

### 6.3 可以自由發揮的項目
- ✅ 情境案例的具體細節（在不違反禁用詞前提下）
- ✅ 互動網頁的視覺設計（在品牌調性內）
- ✅ 腳本的遣詞用字（在調性規範內）

### 6.4 不確定時的處理方式
如果 Claude Code 遇到 Spec 或 Input 沒定義的情境：
1. **不要自己決定**
2. 在輸出檔案最上方加 `## ⚠️ 需人工確認` 區塊
3. 列出具體問題，繼續其他可執行部分
