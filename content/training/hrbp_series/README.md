## 內容設計邏輯

### 為什麼拆四集而不是一支大影片？

| 原因 | 說明 |
|------|------|
| 學習節奏 | 新人 20 分鐘上限，超過記不住 |
| 重用方便 | 忘了怎麼回業績壓力 → 只回看 EP4，不用重刷一小時 |
| 更新成本 | 抽成改了只重做 EP1，心法改了只重做 EP3 |
| 對應實戰節奏 | Day 1 看 EP1+EP2（制度），Day 2 看 EP3+EP4（心法） |

### 為什麼用「混合版」（虛擬人 + Vincent 原聲）？

| 角色 | 負責 | 為什麼 |
|------|------|--------|
| 阿凱（虛擬人） | 架構講述、過場、總結 | 可批量生產，未來好更新 |
| Vincent 原聲 | 關鍵心法金句 | 有溫度、可信度、傳承感 |

### 為什麼互動測驗情境化？

不是「背誦題」，是「你現在正在通電話 → 你會怎麼回？」的情境選擇題。  
每題都附「口袋解析」解釋**為什麼正確答案是對的**，這是真正的學習點。

## 技術規格

### 互動網頁

- **單檔 HTML**（無外部依賴）
- 響應式（手機/平板/桌機）
- **無 localStorage**（Anthropic 環境規則 + 內部平台整合友善）
- **postMessage 通知**：完成測驗會發 `postMessage({ type: 'unit_complete', ...})` 給父視窗，內部平台可接此訊號記錄進度
- 配色統一：主色 `#1E3A5F`（深藍）、強調色 `#F59E0B`（暖橘）

### 腳本格式

- 每集都有 `剪輯資源對照表`（Vincent 原聲時間碼 → 腳本段落）
- 對話區塊用 `[場景]` / `[阿凱]` / `[切 Vincent 原聲]` / `[字幕]` / `[圖卡]` 五種標記
- 圖卡內容直接可做（已寫成可複製的文字塊）
